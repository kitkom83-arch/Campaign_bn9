รวม Widgets สำหรับฝังเว็บ

หน้านี้รวมตัวอย่าง Telegram + X ไว้ที่เดียว เพื่อให้ดูง่ายว่าแต่ละ widget เอาไว้ทำอะไร และแก้โค้ดตรงไหน

Telegram Post Telegram Share Discussion X / Twitter
วิธีใช้หน้านี้: เปลี่ยนเฉพาะค่าที่มีคำว่า แก้ตรงนี้ ก่อน ยังไม่ต้องยุ่งกับระบบ Login จนกว่าจะมี backend.

1) Telegram Widgets

ใช้จริงก่อน ฝังโพสต์

Telegram Post Widget

ใช้โชว์โพสต์จาก public channel หรือ public group ในหน้าเว็บ เช่น t.me/telegram/81 ให้แปลงเป็น telegram/81.

แก้ตรงนี้: data-telegram-post="telegram/81" เป็น ชื่อช่อง/เลขโพสต์
<script async src="https://telegram.org/js/telegram-widget.js?22"
  data-telegram-post="telegram/81"  <!-- แก้ตรงนี้ -->
  data-width="100%"
  data-dark="1">
</script>
ใช้จริงก่อน ปุ่มแชร์

Telegram Share Button

ใช้ทำปุ่มให้คนกดแชร์ลิงก์เว็บหรือข้อความเข้า Telegram ได้ทันที

แก้ตรงนี้: url เป็นลิงก์เว็บพี่ และ text เป็นข้อความที่อยากให้แชร์
แชร์ไป Telegram เปิดช่อง @bn9game 
<a href="https://t.me/share/url?url=https%3A%2F%2Fbn9.one&text=ข้อความที่จะแชร์"
   target="_blank" rel="noopener">
  แชร์ไป Telegram
</a>
เก็บไว้ขั้นต่อไป คอมเมนต์

Telegram Discussion Widget

ใช้ฝังคอมเมนต์/การสนทนาจากโพสต์ Telegram ที่เปิด comments แล้ว เหมาะกับหน้าบทความหรือหน้าแคมเปญที่อยากให้คนคุยกัน

ต้องใช้โพสต์ที่มี comments หรือ channel ที่ตั้ง discussion group ไว้ก่อน
<script async src="https://telegram.org/js/telegram-widget.js?22"
  data-telegram-discussion="telegram/81"  <!-- แก้ตรงนี้ -->
  data-comments-limit="5"
  data-height="360"
  data-dark="1">
</script>
ยังไม่ทำตอนนี้ Login

Telegram Login

ตัวนี้ไม่ใช่แค่วางโค้ดแล้วจบ ต้องมี BotFather, Client ID, Allowed URLs และต้องตรวจ token ฝั่ง server ก่อนเชื่อข้อมูลผู้ใช้

สำหรับเว็บ static หน้าเดียว ให้ทำเป็นปุ่ม placeholder ก่อน อย่าเปิด Login จริงจนกว่าจะมี backend
Telegram Login: รอ Backend 
// ขั้นตอนจริงภายหลัง
1. สร้าง/เลือก Bot ใน BotFather
2. ตั้ง Allowed URLs
3. ได้ Client ID + Client Secret
4. ฝัง Login library
5. backend ต้อง verify id_token ก่อนใช้งาน

2) X / Twitter Widgets

ใช้จริงก่อน Timeline

X Profile Timeline

ใช้โชว์ฟีดโพสต์จากบัญชี X แบบ public timeline

แก้ตรงนี้: เปลี่ยน https://x.com/XDevelopers เป็นบัญชี X ของพี่
Posts by XDevelopers 
<a class="twitter-timeline"
   data-theme="dark"
   data-height="420"
   href="https://x.com/XDevelopers">
  Posts by XDevelopers
</a>
ใช้จริงก่อน Share/Post

X Share Button

ใช้ทำปุ่มให้คนกดโพสต์ข้อความพร้อมลิงก์ไป X

แก้ตรงนี้: text และ url
แชร์ไป X 
<a href="https://twitter.com/intent/tweet?text=ข้อความ&url=https%3A%2F%2Fbn9.one"
   target="_blank" rel="noopener">
  แชร์ไป X
</a>
ฝังโพสต์เดี่ยว

X Single Post Embed

ใช้ฝังโพสต์ X เดี่ยว ถ้ามีลิงก์โพสต์จริง ให้เอาลิงก์มาใส่ใน href

ตอนนี้ใส่ตัวอย่างโครงไว้ก่อน พี่ส่งลิงก์โพสต์จริงมาแล้วค่อยเปลี่ยน
<blockquote class="twitter-tweet" data-theme="dark">
  <a href="https://x.com/USERNAME/status/POST_ID"></a>
</blockquote>

3) ตารางสรุปว่าใช้ตัวไหนก่อน

Widget ใช้ทำอะไร สถานะ จุดที่ต้องแก้
Telegram Post ฝังโพสต์ Telegram ทำก่อน data-telegram-post="channel/postId"
Telegram Share ปุ่มแชร์ลิงก์ไป Telegram ทำก่อน url และ text
Telegram Discussion ฝังคอมเมนต์จากโพสต์ ทำทีหลัง data-telegram-discussion
Telegram Login ล็อกอินด้วย Telegram รอ backend Client ID / Allowed URLs / Verify token
X Timeline โชว์ฟีด X จากบัญชี public ทำก่อน href="https://x.com/USERNAME"
X Share ปุ่มโพสต์แชร์ไป X ทำก่อน text และ url
